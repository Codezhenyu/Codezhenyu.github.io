构造器

定义及意义：

规则：

  1. 在对象创建时候自动调用，完成初始化相关工作。
  2. 无返回值，与类名同。
  3. 可以重载，可以默认参数。
  4. 默认无参空体，一经实现，默认不复存在。

析造器

对象销毁时期：

  1. 栈对象离开其作用域。
  2. 堆对象被手动delete

析构器的定义及意义：

在类对象销毁时，自动调用，完成对象的销毁。尤其是类中已申请的堆内存的释放.

规则：

  1. 对象销毁时，自动调用。完成销毁的善后工作。
  2. 无返回值，与类名同，无参。不可以重载与默认参数。
  3. 系统提供默认空析构器，一经实现，不复存在。

小结 ：

析构函数的作用，并不是删除对象，而在对象销毁前完成的一些清理工作。

多文件编程

通常我们将类的声明，放到.h文件中去，而将实现放在.cpp中去。

拷贝构造(Copy contructor)

拷贝构造的定义及意义：

由已经存在的对象，创建新对象。也就是说新对象，不由构造器来构造，而是 由拷贝构造器来完成。拷贝构造器的格式是固定的。

规则：

  1. 系统提供默认的拷贝构造器。一经实现，不复存在。
  2. 系统提供时等位拷贝，也就是所谓的浅浅的拷贝。
  3. 要实现拷贝，必须要自定义。

拷贝构造发生的时机：

  1. 制作对象的副本。
  2. 以对象作为参数和返回值。

深拷贝与浅拷贝：

系统提供默认的拷贝构造器，一经定义不再提供。但系统提供的默认拷贝构造器是等位
拷贝，也就是通常意义上的浅拷贝。如果类中包含的数据元素全部在栈上，浅拷贝也可以满
足需求的。但如果堆上的数据，则会发生多次析构行为。
